# Sistema_Bancario_POO_Dio
## Desafio Dio - Suzano - Python Developer
Este projeto consiste em um sistema bancário completo que integra o paradigma funcional com a Programação Orientada a Objetos (POO). O simulador permite o cadastro de usuários, criação de contas e operações financeiras com regras de negócio dinâmicas.
Arquitetura POO: Implementação de classes abstratas com a biblioteca abc, garantindo contratos rigorosos para transações (Saque e Depósito).
Hierarquia de Classes: Uso de herança para especialização de contas e clientes.
Funcionalidades Modernas: Aplicação de argumentos estritamente posicionais (/) e nomeados (*), demonstrando controle preciso sobre a interface das funções.
Persistência de Dados (Em Memória): Gerenciamento de múltiplos usuários e contas vinculadas através de coleções estruturadas.
